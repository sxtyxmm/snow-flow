Installation & Setup

Install via NPM

# Install Snow-Flow globally
npm install -g snow-flow

# Initialize Snow-Flow in your project
snow-flow init

# Authenticate with ServiceNow
snow-flow auth login

Install from GitHub

# Clone the repository
git clone https://github.com/groeimetai/snow-flow.git

# Navigate to directory
cd snow-flow

# Install dependencies
npm install

# Build the project
npm run build

# Link globally
npm link

# Initialize
snow-flow init

Configuration

// .snow-flow.config.json
{
  "instance": "https://your-instance.service-now.com",
  "auth": {
    "type": "oauth",
    "clientId": "your-client-id",
    "clientSecret": "your-client-secret"

  }
}

Deploy a Dashboard Widget

// Create and deploy an incident dashboard widget
await snow_deploy_widget({
  name: "Incident Dashboard",
  template: `
    <div class="dashboard">
      <div ng-repeat="incident in data.incidents">
        <span>{{incident.number}}</span>
        <span>{{incident.short_description}}</span>
      </div>
    </div>
  `,
  server_script: `
    (function() {
      var gr = new GlideRecord('incident');
      gr.addActiveQuery();
      gr.query();
      data.incidents = [];
      while(gr.next()) {
        data.incidents.push({
          number: gr.getValue('number'),
          short_description: gr.getValue('short_description')
        });
      }
    })();
  `,
  client_script: `
    function($scope) {
      var c = this;
      // Client logic here
    }
  `
});

Execute Background Script

// Execute script with full output capture
const result = await snow_execute_script_with_output({
  script: `
    gs.info('Starting incident analysis...');
    
    var stats = {};
    var gr = new GlideRecord('incident');
    gr.addActiveQuery();
    gr.query();
    
    stats.total = gr.getRowCount();
    
    var priority = new GlideAggregate('incident');
    priority.addActiveQuery();
    priority.groupBy('priority');
    priority.addAggregate('COUNT');
    priority.query();
    
    stats.byPriority = {};
    while(priority.next()) {
      var p = priority.getValue('priority');
      stats.byPriority[p] = priority.getAggregate('COUNT');
    }
    
    gs.print('Analysis complete: ' + JSON.stringify(stats));
    return stats;
  `
});

console.log('Execution Output:', result.output);
console.log('Returned Data:', result.returnValue);

Train Incident Classifier

// Train ML model for incident classification
const model = await snow_train_classifier({
  type: 'incident_category',
  training_data: {
    table: 'incident',
    fields: ['short_description', 'description'],
    label: 'category',
    limit: 1000
  },
  model_config: {
    type: 'lstm',
    epochs: 50,
    batch_size: 32
  }
});

// Use the trained model
const prediction = await snow_predict({
  model_id: model.id,
  input: {
    short_description: "Cannot access email",
    description: "Outlook is not connecting to server"
  }
});

console.log('Predicted category:', prediction.category);
console.log('Confidence:', prediction.confidence);

Best Practices

üîç Always Verify First

Never assume something doesn't exist. Always verify with background scripts before making changes.

üìù Use ES5 JavaScript

ServiceNow uses Rhino engine. Always write ES5-compatible code - no arrow functions, const/let, or template literals.

üîÑ Sync User Changes

When users modify artifacts in ServiceNow, always fetch the latest version before making additional changes.

‚úÖ Widget Coherence

Ensure perfect communication between HTML template, client script, and server script in widgets.

üìä Use Real Data

Snow-Flow never uses mock data. Always test with real ServiceNow instance data.

üöÄ Leverage MCP Servers

Use specialized MCP servers for specific tasks - each is optimized for its domain.
