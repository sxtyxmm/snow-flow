üîë ServiceNow OAuth Setup Guide

1. Navigate to System OAuth ‚Üí Application Registry

In ServiceNow navigator, search for "Application Registry"

2. Click "New" ‚Üí "Create an OAuth API endpoint"

Choose the endpoint option, not client credentials

3. Fill Required Fields:
  • ‚Ä¢ Name: Snow-Flow Integration
  • ‚Ä¢ Client ID: snow-flow-client
  • ‚Ä¢ Redirect URL: http://localhost:3000/callback
4. Save and Copy Credentials

Copy the Client ID and Client Secret - you'll need these for authentication

5. Run Snow-Flow Authentication
snow-flow auth login

This will open browser, enter your ServiceNow credentials, and save tokens automatically

üí° OAuth Token Management
  • ‚Ä¢ Automatic Refresh: Snow-Flow handles token refresh automatically
  • ‚Ä¢ Secure Storage: Tokens stored locally in encrypted format
  • ‚Ä¢ Expiration Handling: Clear error messages when tokens expire
  • ‚Ä¢ Re-authentication: Simply run snow-flow auth login again

üîß Troubleshooting Guide

Common Issues

üîê OAuth Token Expired - "Could not find artifact"

Error: "Could not find artifact with sys_id xyz... It may be in a custom table or you may lack permissions."

Real Cause: Your ServiceNow OAuth token has expired, not missing artifact!

# SOLUTION: Re-authenticate snow-flow auth login

Fixed in v4.5.3: Now shows clear OAuth error instead of misleading "artifact not found"

üì° Stdio Connection Errors

Error: Cannot connect to stdio/MCP servers

Cause: Claude Code not running or wrong startup order

# SOLUTION: Install and start Claude Code FIRST npm install -g @anthropic-ai/claude-code claude login cd /your/project claude --dangerously-skip-permissions # THEN start Snow-Flow snow-flow init
‚ùå Permission Errors in Claude Code

"dangerously skip permissions" error

# SOLUTION: Login first! claude login claude --dangerously-skip-permissions
‚ùå ES5 Syntax Errors

ServiceNow uses Rhino engine - ES6+ fails

# ‚ùå WRONG: Modern JavaScript const users = data.filter(u => u.active); # ‚úÖ CORRECT: ES5 Only var users = []; for (var i = 0; i < data.length; i++) { if (data[i].active) users.push(data[i]); }

Debug Commands

# Test ServiceNow connection snow-flow auth login # Verify authentication status snow-flow auth status # Re-initialize if needed snow-flow init # Check version and help snow-flow --version snow-flow --help # Enable verbose logging export DEBUG=snow-flow:*

Quick Fixes

  • MCP not responding: Re-run snow-flow init and restart Claude Code
  • Auth failed: Run snow-flow auth login again
  • Widget deployment failed: Check ES5 syntax and field names
  • Claude permission errors: Always run claude login first
